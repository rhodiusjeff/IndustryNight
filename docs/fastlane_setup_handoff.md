# Fastlane iOS Setup — Agent Handoff Context

This document captures the complete Fastlane configuration used in **vmap_app_barebones** so another agent or developer can replicate it for a new Flutter app.

---

## Overview

Fastlane is used to build the Flutter iOS app and upload it to **TestFlight** using an **App Store Connect API key** (no username/password, no interactive auth). Two lanes exist:

| Lane | Purpose |
|---|---|
| `preflight` | Sanity-check build (no codesigning) |
| `beta` | Full release build → upload to TestFlight |

---

## Directory Structure

All Fastlane files live in `ios/fastlane/` relative to the Flutter project root:

```
ios/
  fastlane/
    Appfile          # App identifier + team ID
    Fastfile         # Lane definitions
    .env             # Secrets (NOT committed to git)
    .env.example     # Template committed to git
    README.md        # Auto-generated by fastlane
```

### 1. Xcode Command Line Tools
```sh
xcode-select --install
```

Install via Bundler (recommended) or Homebrew:
```sh
# Option A — Homebrew
brew install fastlane

# Option B — Bundler (add to ios/Gemfile)
gem install bundler
bundle exec fastlane
```

### 3. Flutter
The lanes call `flutter pub get` and `flutter build ipa` internally, so Flutter must be on `$PATH`.

## App Store Connect API Key

> **This is the only credential needed — no Apple ID password required.**

1. Log in to [App Store Connect → Users and Access → Integrations → App Store Connect API](https://appstoreconnect.apple.com/access/integrations/api).
2. Click **+** to generate a new key. Role: **App Manager** (or Admin).
3. Download the `.p8` file **once** (it cannot be re-downloaded).
4. Note:
   - **Key ID** — 10-character alphanumeric string (e.g. `SU58N7WCW7`)
   - **Issuer ID** — UUID shown at the top of the API Keys page (e.g. `94bca5a5-c92c-4cab-8b96-f133562020cc`)

### Recommended `.p8` storage location
```
~/.appstoreconnect/private_keys/AuthKey_<KEYID>.p8
```
Fastlane also searches this directory automatically if you point `APP_STORE_CONNECT_KEY_FILEPATH` at the directory rather than the file.

## Files to Create

### `ios/fastlane/Appfile`
```ruby
# Fastlane configuration for the iOS app.
# Keep secrets out of this file; prefer environment variables and API keys.

app_identifier("<YOUR_BUNDLE_ID>")        # e.g. com.example.myapp
team_id(ENV["FASTLANE_TEAM_ID"] || "<YOUR_TEAM_ID>")  # 10-char Apple dev portal team ID

# If you need to disambiguate multiple App Store Connect teams, set:
# itc_team_id(ENV["FASTLANE_ITC_TEAM_ID"])
```

### `ios/fastlane/Fastfile`
```ruby
default_platform(:ios)

platform :ios do
  desc "Sanity check version/build/name/bundle id without codesigning"
  lane :preflight do
    sh("cd .. && flutter pub get")
    sh("cd .. && flutter build ipa --release --no-codesign")
  end

  desc "Build and upload a release build to TestFlight"
  lane :beta do
    key_id = ENV.fetch("APP_STORE_CONNECT_KEY_ID")
    issuer_id = ENV.fetch("APP_STORE_CONNECT_ISSUER_ID")

    key_path = ENV.fetch("APP_STORE_CONNECT_KEY_FILEPATH")
    key_path = File.expand_path(key_path)
    if File.directory?(key_path)
      key_path = File.join(key_path, "AuthKey_#{key_id}.p8")
    end
    UI.user_error!(
      "App Store Connect API key file not found at: #{key_path}\n" \
      "Set APP_STORE_CONNECT_KEY_FILEPATH to the full path of AuthKey_<KEYID>.p8 " \
      "or to the directory that contains it."
    ) unless File.exist?(key_path)

    api_key = app_store_connect_api_key(
      key_id: key_id,
      issuer_id: issuer_id,
      key_filepath: key_path,
      duration: 1200,
      in_house: false
    )

    build_app(
      workspace: "Runner.xcworkspace",
      scheme: "Runner",
      configuration: "Release",
      export_method: "app-store",
      clean: true
    )

    upload_to_testflight(
      api_key: api_key,
      changelog: ENV["FL_CHANGELOG"],
      skip_waiting_for_build_processing: true
    )
  end
end
```

### `ios/fastlane/.env.example`  *(commit this)*
```sh
# Copy to ios/fastlane/.env (do not commit) and fill in values.
# Create an App Store Connect API key in App Store Connect → Users and Access → Keys.

# The key ID (10 chars) shown next to your API key.
APP_STORE_CONNECT_KEY_ID=

# The issuer ID (UUID) shown in the API keys page.
APP_STORE_CONNECT_ISSUER_ID=

# Path to the downloaded .p8 key file (preferred), OR the directory containing it.
# Recommended file location:
#   ~/.appstoreconnect/private_keys/AuthKey_<KEYID>.p8
# Example (full file path):
#   /Users/<you>/.appstoreconnect/private_keys/AuthKey_<KEYID>.p8
# Example (directory):
#   /Users/<you>/.appstoreconnect/private_keys
APP_STORE_CONNECT_KEY_FILEPATH=

# Optional: dev portal team id (10 chars). Defaults to the value in Appfile.
FASTLANE_TEAM_ID=

# Optional: release notes shown in TestFlight.
FL_CHANGELOG=
```

### `ios/fastlane/.env`  *(never commit — add to .gitignore)*
```sh
APP_STORE_CONNECT_KEY_ID=<YOUR_KEY_ID>
APP_STORE_CONNECT_ISSUER_ID=<YOUR_ISSUER_UUID>
APP_STORE_CONNECT_KEY_FILEPATH=/Users/<you>/.appstoreconnect/private_keys/AuthKey_<YOUR_KEY_ID>.p8
FASTLANE_TEAM_ID=<YOUR_TEAM_ID>
FL_CHANGELOG=<version> TestFlight build
```

## Running the Lanes

All commands are run from the `ios/` directory:

```sh
cd ios

# Sanity check (no upload, no codesigning)
bundle exec fastlane ios preflight

# Build + upload to TestFlight
bundle exec fastlane ios beta
```

Or without Bundler:
```sh
fastlane ios beta
```

## Notes for the Receiving Agent

- The `build_app` action uses `Runner.xcworkspace` — standard for all Flutter iOS projects.
- `flutter build ipa` is called via `sh()` inside the lane, not via a fastlane action, because Flutter manages its own build toolchain.
- `skip_waiting_for_build_processing: true` is set on `upload_to_testflight` to avoid the lane hanging for 10–20 minutes after upload. The build will appear in TestFlight once Apple finishes processing.
- The `preflight` lane is useful in CI to catch version/bundle-ID mismatches before doing a full codesigned build.
- Codesigning is handled automatically by Xcode's automatic signing (no `match` / `sigh` needed for this setup).
