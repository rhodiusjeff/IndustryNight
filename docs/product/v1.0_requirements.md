# v1.0 Requirements — Pending for App Store Launch

This document tracks features and polish items deferred from the initial v0.5 release that must be completed before a 1.0 App Store submission.

## Current version: 0.5.0
## Target: 1.0.0

---

## Social App

### Profile Photo Upload
- **What:** Users can set a profile photo from their camera roll
- **Why deferred:** Requires image picker + crop/fit UX (circular frame) — not trivial to do well
- **Scope:**
  - Re-add `image_picker` to pubspec.yaml
  - Implement circular crop/reposition UI (fit-to-circle) in `edit_profile_screen.dart`
  - Wire to existing `AppState.uploadProfilePhoto()` (API already supports this)
  - Add `NSPhotoLibraryUsageDescription` to `ios/Runner/Info.plist`
  - Add `NSPhotoLibraryAddUsageDescription` if saving back to camera roll
  - Handle onboarding flow: prompt for photo during `profile_setup_screen`
- **Files:** `edit_profile_screen.dart`, `profile_setup_screen.dart`, `app_state.dart`, `Info.plist`, `pubspec.yaml`

### App Store Assets
- **What:** Required before submission
- **Scope:**
  - App Store screenshots: 6.5" iPhone (required), 5.5" iPhone (required), 12.9" iPad (optional but recommended)
  - App Store description and keywords
  - Support URL (required — must be a live URL)
  - Privacy policy URL (required — app collects phone numbers and location-adjacent data)
  - Age rating questionnaire answers
- **Notes:** Screenshots must show real app content, not placeholder data

### App Icon — Alpha Channel
- **What:** 1024x1024 App Store icon must NOT have an alpha channel (Apple rejects RGBA)
- **Why deferred:** Requires design tool export change
- **Scope:** Re-export `Icon-App-1024x1024@1x.png` as RGB (no alpha) from source design file
- **Files:** `ios/Runner/Assets.xcassets/AppIcon.appiconset/Icon-App-1024x1024@1x.png`

### Orientation Lock
- **Decision pending:** Currently supports Portrait + Landscape. For a mobile-first social app, Portrait-only is conventional. Decide and lock before 1.0.
- **Files:** `ios/Runner/Info.plist` (`UISupportedInterfaceOrientations`)

### Push Notifications
- **What:** QR connection notifications currently use 4-second polling. Push would replace polling for reliability.
- **Scope:** APNs entitlement, Firebase/APNs setup, backend notification dispatch on new connection
- **Notes:** Polling works for v0.5; push is a quality-of-life improvement for 1.0

## Account Deletion (App Store Requirement)
- **What:** Apple requires all apps that allow account creation to offer in-app account deletion
- **Status:** `DELETE /auth/me` API endpoint exists. UI may or may not be exposed.
- **Scope:** Verify `settings_screen.dart` has a visible "Delete Account" option that calls the API
- **Files:** `settings_screen.dart`, `app_state.dart`
